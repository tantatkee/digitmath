# DigitMath — Implementation Plan

## Overview

DigitMath is a mobile-first math equation-building game targeting elementary and middle school children. Players are given N single-digit numbers (where N = 4, 5, or 6 depending on difficulty) and must form a valid mathematical equation using all the numbers and any arithmetic operators (+, −, ×, ÷, (, )).

The app is built with **Vite + React** for the web UI and **Capacitor** for wrapping it as a native Android and iOS app.

## Game Modes

### Single Player
- 10 rounds per session
- 10 points per correct answer
- Time penalty at the end: subtract `floor(totalSeconds / 600)` from score (1 point per 10 minutes)
- Player may skip any round (no penalty, no points)

### Multiplayer (2–4 players, hot-seat)
- 10 rounds per player per session (total rounds = 10 × numPlayers)
- Players take turns on the same device
- Each correct answer: **10 + bonus** points
- If a player skips, the **same puzzle** is passed to the next player with an additional **+1 bonus point** per skip
- On a correct solve after a skip chain, the bonus resets to 0
- **Time penalty tracked separately per player** — only the current player's elapsed time and penalty is shown during their turn
- Each player's penalty = `floor(totalTurnSeconds / 30)`, deducted from their score at the end (1 point per 30 seconds)
- Leaderboard shows base score, penalty, and net score per player

## Puzzle Engine (`src/utils/engine.js`)

Guarantees that every generated puzzle has at least one valid solution.

### Algorithm
1. Randomly generate N integers from 1–9 (0 excluded to avoid trivial `x * 0 = 0`).
2. Try all permutations of the digits.
3. For each permutation, try all split points (LHS count = i, RHS count = N−i).
4. For each split, try all operator combinations on both sides using `+`, `−`, `×`, `÷`.
5. Evaluate each pair; if LHS == RHS within floating-point tolerance (1e-6), a solution exists.
6. Return the digits if a solution is found; otherwise retry (up to 500 attempts).

### Performance
| N | Evaluation space | Typical time |
|---|-------------------|--------------|
| 4 | ~384 checks | < 5ms |
| 5 | ~6,144 checks | < 50ms |
| 6 | ~92,160 checks | < 250ms |

### Equation Verification (`verifyUserExpression`)
Validates a player's input as an array of tokens (numbers and operators):
1. Must contain exactly one `=` token.
2. The set of digits used must match the puzzle digits exactly.
3. Evaluates LHS and RHS using safe `new Function()` and checks equality.

## Sound Effects (`src/utils/sounds.js`)

All sounds are generated at runtime using the **Web Audio API** — no external audio files needed.

| Event | Sound | Description |
|-------|-------|-------------|
| Block tap | `playClick()` | Short square-wave tick |
| Undo / Clear | `playUndo()` | Lower triangle-wave |
| Correct answer | `playCorrect()` | Ascending C–E–G–C arpeggio |
| Wrong answer | `playWrong()` | Buzzy sawtooth error tone |
| Skip | `playSkip()` | Descending frequency sweep |
| Game over | `playGameOver()` | 5-note triumphant fanfare |

## UI Design System (`src/index.css`)

- **Font:** [Fredoka](https://fonts.google.com/specimen/Fredoka) — round, playful, highly legible
- **Color palette:** vibrant blues, yellow, green, red, purple (Kahoot/Blooket style)
- **Button style:** large rounded corners, thick colored drop-shadows (`box-shadow: 0 6px 0 darkerColor`)
- **Layout:** `100dvh` (dynamic viewport height) to prevent scrolling on mobile browsers; `overflow: hidden` on `#root`
- **Micro-interactions:** buttons push down 4px on `:active` with shadow collapse

| iOS | `ios/` | Xcode + CocoaPods |

### Known Gradle issue (fixed)
Capacitor + older plugins may pull in conflicting Kotlin stdlib versions. The fix in `android/build.gradle` forces all modules to use `kotlin-stdlib:1.8.22`:
```gradle
configurations.all {
    resolutionStrategy {
        force 'org.jetbrains.kotlin:kotlin-stdlib:1.8.22'
        force 'org.jetbrains.kotlin:kotlin-stdlib-jdk7:1.8.22'
        force 'org.jetbrains.kotlin:kotlin-stdlib-jdk8:1.8.22'
    }
}
```
